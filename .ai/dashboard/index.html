Pipeline timeline

Each row is one Claude Code session (a single chat window's worth of work). Bars are subprocess dispatches captured by the PostToolUse hook in .ai/dashboard/log_event.py: claude -p ... and codex exec ... calls. The task line is the first user message of that session; (no transcript) means the session file isn't on disk anymore. Bars labelled (ad-hoc) were model calls that didn't match a formal pipeline phase pattern. Inline-mode phases (when the orchestrator runs a phase in its own session without dispatching) are not captured — they leave no subprocess trace.
success (exit 0) failure (non-zero) pending / unknown exit ad-hoc (phase not detected)

Auto-select rankings

Ranking per (phase, size, risk, budget) tuple, computed from .ai/metrics.jsonl (the orchestrator appends one line per dispatched phase). Candidates with fewer than 3 samples are excluded; groups with no qualifying candidates are omitted. The planner's adaptive scorer ranks with the same formula: 0.6·success_rate + 0.2·(1-norm_duration) + 0.2·budget_align. When the planner falls back to the static table (cold-start or guard rail), the line in ## Selected models annotates reason="static: …".

Auto-improver

Reads/writes the improver: block in .ai/models.yaml. Controls the background skill-improvement loop.

Auto-select

Reads/writes the auto_select: block. When enabled, the planner picks tool/model per phase based on the task.

Per-phase fallback

Fallback values written to .ai/models.yaml. Used when auto-select is off, or when the planner has no auto-select match for a phase. reasoning_effort maps to claude --effort (low/medium/high/xhigh/max) and to codex --config model_reasoning_effort (low/medium/high/xhigh — no max). timeout_seconds overrides the default (1800s for execute, 600s for others); empty = default.
