Phase routing

Which tool / model handles each phase, plus the runtime knobs (reasoning_effort, timeout_seconds). Click Edit on a row to change any column.

Inline-when-possible: phase runs in the current session if the tool / model match, otherwise as a fresh subprocess. Always-subprocess: every phase starts a fresh subprocess regardless.

Auto-select

When enabled, the planner picks tool / model per phase based on the task's (size, risk, budget) using the rankings below. When disabled, the Phase routing table above is used as-is.

Auto-select rankings

Telemetry-driven scores backing the planner's per-task pick. Read-only.

How auto-select ranking works
Ranking per (phase, size, risk, budget) tuple, computed from .ai/ledgers/metrics.jsonl (the orchestrator appends one line per dispatched phase). Candidates with fewer than 3 samples are excluded; groups with no qualifying candidates are omitted. The planner's adaptive scorer ranks with the same formula: 0.6·success_rate + 0.2·(1-norm_duration) + 0.2·budget_align. When the planner falls back to the static table (cold-start or guard rail), the line in ## Selected models annotates reason="static: …".
